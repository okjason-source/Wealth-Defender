# Contributing to Wealth Defender

Thank you for your interest in contributing to Wealth Defender! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/wealth-defender.git`
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature-name`

## Development Workflow

1. Make your changes
2. Test your changes: `npm run dev`
3. Build to verify: `npm run build`
4. Commit your changes with clear messages
5. Push to your fork: `git push origin feature/your-feature-name`
6. Open a Pull Request

## Code Style

- Use TypeScript for all new code
- Follow existing code style and patterns
- Add comments for complex logic
- Keep functions focused and modular

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add new feature`
- `fix: Fix bug description`
- `docs: Update documentation`
- `refactor: Refactor code`
- `style: Format code`

## Pull Request Process

1. Ensure your code builds without errors
2. Test the game thoroughly
3. Update documentation if needed
4. Describe your changes clearly in the PR

## Questions?

Open an issue for questions or discussions about contributions.
